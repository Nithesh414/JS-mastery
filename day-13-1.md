# Day 13 - Leader Elements in an Array

## Problem Statement
Given an array, find all **leader elements**.

A **leader** is an element that is greater than all the elements to its right side.

### Example
Input:
```js
[16, 17, 4, 3, 5, 2]
```
Output:
```js
[17, 5, 2]
```

Explanation:
- `17` is greater than `4, 3, 5, 2`
- `5` is greater than `2`
- `2` is the last element, so it is always a leader

---

## 1) Brute Force Approach - O(n²)

### Code
```js
let input = [16,17,4,3,5,2];
let lead = [];

for (let i = 0; i < input.length; i++) {
    let islead = true;

    for (let j = i + 1; j < input.length; j++) {
        if (input[i] < input[j]) {
            islead = false;
            break;
        }
    }

    if (islead) {
        lead.push(input[i]);
    }
}

console.log(lead);
```

### Explanation
This method checks each element one by one.

- Take one element at index `i`
- Compare it with every element to its right
- If any right-side element is greater, then it is **not** a leader
- If no greater element is found, then it is a leader

### Dry Run
For `16`:
- Compare with `17, 4, 3, 5, 2`
- Since `17` is greater than `16`, `16` is not a leader

For `17`:
- Compare with `4, 3, 5, 2`
- No element is greater than `17`
- So `17` is a leader

For `4`:
- Compare with `3, 5, 2`
- Since `5` is greater, `4` is not a leader

For `3`:
- Compare with `5, 2`
- Since `5` is greater, `3` is not a leader

For `5`:
- Compare with `2`
- No element is greater, so `5` is a leader

For `2`:
- No elements on the right
- So `2` is a leader

Output:
```js
[17, 5, 2]
```

### Time Complexity
- Outer loop runs `n` times
- Inner loop may also run up to `n` times
- So total time complexity is **O(n²)**

### Space Complexity
- Result array stores leader elements
- So space complexity is **O(n)**

---

## 2) Optimized Approach - O(n)

### Code
```js
let input = [16,17,4,3,5,2];
let lead = [];

let maxRight = input[input.length - 1];
lead.push(maxRight);

for (let i = input.length - 2; i >= 0; i--) {
    if (input[i] > maxRight) {
        lead.push(input[i]);
        maxRight = input[i];
    }
}

lead.reverse(); // to maintain original order
console.log(lead);
```

### Explanation
This method is more efficient.

### Key Idea
Instead of checking every element with all elements to the right:
- Start from the **rightmost element**
- Keep track of the **maximum element seen so far** using `maxRight`
- If the current element is greater than `maxRight`, then it is a leader

### Why this works
A leader must be greater than all elements on its right.
If we move from right to left, we only need to remember the largest element seen so far.

### Dry Run
Input:
```js
[16, 17, 4, 3, 5, 2]
```

- Start with last element `2`
  - Last element is always a leader
  - `lead = [2]`
  - `maxRight = 2`

- Move left to `5`
  - `5 > 2`, so `5` is a leader
  - `lead = [2, 5]`
  - `maxRight = 5`

- Move left to `3`
  - `3 < 5`, not a leader

- Move left to `4`
  - `4 < 5`, not a leader

- Move left to `17`
  - `17 > 5`, so `17` is a leader
  - `lead = [2, 5, 17]`
  - `maxRight = 17`

- Move left to `16`
  - `16 < 17`, not a leader

Since leaders were collected from right to left, reverse the array:
```js
[17, 5, 2]
```

### Time Complexity
- Single loop runs through the array once
- So time complexity is **O(n)**

### Space Complexity
- Result array stores leader elements
- So space complexity is **O(n)**

---

## Comparison Table

| Approach | Method | Time Complexity | Space Complexity |
|----------|--------|-----------------|------------------|
| Brute Force | Compare each element with all right-side elements | O(n²) | O(n) |
| Optimized | Traverse from right and track maximum | O(n) | O(n) |

---

## Conclusion
- The **brute force approach** is easy to understand, but slower for large arrays
- The **optimized approach** is better because it reduces the time complexity from **O(n²)** to **O(n)**
- For interviews and real projects, the **optimized approach** is preferred
